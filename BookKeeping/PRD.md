# 个人记账应用 PRD

**文档版本：** v1.0  
**状态：** 草案（待确认）  
**日期：** 2026-07-21

---

## 1. 产品概述和目标用户

### 1.1 产品概述

一款极简个人记账 Web 应用，帮助用户在 10 秒内记下一笔收支，并按日查看、按月统计。刻意不做预算、账本、多币种、社交等复杂能力，降低使用门槛，服务「养成记账习惯」。

### 1.2 产品目标

| 目标 | 说明 |
|------|------|
| 记得出 | 记录路径短，主路径 ≤ 3 步完成一笔记账 |
| 看得懂 | 当日流水清晰，当月收支与分类占比一目了然 |
| 用得稳 | 数据本地持久化，刷新/重开不丢账 |
| 学得会 | 无教程也能上手，界面信息密度低 |

### 1.3 目标用户

- **主要用户：** 想养成记账习惯的个人用户（学生、上班族）
- **使用场景：** 付款后立刻记一笔（手机）；周末/月末看月报（电脑）
- **非目标用户：** 需要多账本、共享家庭账、发票报销、企业财务的人群

### 1.4 成功标准（MVP）

- 用户能在 1 分钟内完成：记一笔 → 在当日列表看到 → 在当月统计看到汇总
- 核心操作无需说明书
- 数据仅存本机，不依赖登录即可使用

### 1.5 产品形态

- **响应式 Web 应用**：手机 + 电脑同一套，自适应布局
- **数据本地存储**：无需账号即可使用

## 2. 详细功能列表

### 2.1 记账（核心）

| 功能 | 描述 |
|------|------|
| 新增记录 | 选择收入/支出、金额、分类、日期（默认今天）、可选备注 |
| 编辑记录 | 修改已有记录的全部字段 |
| 删除记录 | 删除前二次确认 |
| 金额校验 | 金额 > 0；最多 2 位小数；禁止非法字符 |

**单条记录字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识 |
| type | `income` \| `expense` | 是 | 收入 / 支出 |
| amount | number | 是 | 金额（元） |
| category | string | 是 | 分类 ID 或名称 |
| date | string (YYYY-MM-DD) | 是 | 记账日期 |
| note | string | 否 | 备注，建议上限 100 字 |
| createdAt / updatedAt | ISO 时间 | 是 | 系统维护 |

### 2.2 分类（预设，MVP 不可自定义）

**支出预设：** 餐饮、交通、购物、居住、娱乐、医疗、教育、其他  
**收入预设：** 工资、兼职、红包、投资、其他  

分类与类型绑定：选「支出」只出现支出分类，选「收入」只出现收入分类。

### 2.3 流水查看

| 功能 | 描述 |
|------|------|
| 按日查看 | 默认展示今天；可切换前后日期或选日期 |
| 日汇总 | 当日收入合计、支出合计、结余（收−支） |
| 记录列表 | 按时间倒序；展示类型、分类、金额、备注摘要 |
| 空状态 | 当日无记录时给出友好提示 + 快捷「记一笔」入口 |

### 2.4 月度统计

| 功能 | 描述 |
|------|------|
| 月份切换 | 上一月 / 下一月 / 回到本月 |
| 月汇总 | 总收入、总支出、结余 |
| 分类占比 | 支出（及可选收入）按分类汇总；可用简单条形/环形图 |
| 月内流水入口 | 从统计可跳到该月某日或该月流水列表（MVP 可仅支持「按日」+「按月汇总」） |

### 2.5 数据与设置（MVP 最小集）

| 功能 | 描述 |
|------|------|
| 本地持久化 | 所有记录存浏览器本地，刷新不丢 |
| 导出（可选进后续） | JSON/CSV 导出备份 |
| 清空数据 | 危险操作，需明确确认（可放后续） |

### 2.6 明确不做（YAGNI）

- 登录 / 云同步 / 多设备实时同步
- 多账本、多币种、预算、借贷、分期
- 账单图片 OCR、银行导入
- 社交分享、会员体系

## 3. 功能优先级

### 3.1 MVP（v1.0）— 必须有

1. 新增 / 编辑 / 删除收支记录
2. 预设分类（收支分套）
3. 按日查看流水 + 日汇总
4. 月度汇总（总收、总支、结余）+ 支出分类占比
5. 本地存储持久化
6. 响应式布局（手机主用 + 电脑可用）

**MVP 验收口诀：** 能记、能改、能删、能按日看、能按月看、刷新不丢。

### 3.2 后续版本

| 版本 | 功能 | 价值 |
|------|------|------|
| v1.1 | 备注搜索 / 按分类筛选当日或当月 | 找账更快 |
| v1.1 | 月度简易图表交互（点击分类看明细） | 统计更可操作 |
| v1.2 | 导出 JSON/CSV、导入备份 | 换机/备份 |
| v1.2 | 自定义分类（增删改、排序） | 灵活度 |
| v1.3 | PWA（加到主屏幕、离线壳） | 更接近 App |
| 更远 | 可选云备份（需账号） | 多设备，超出当前「本地」原则时再议 |

## 4. 界面设计要求

### 4.1 信息架构（3 个主界面即可）

1. **首页 / 今日** — 日汇总 + 当日列表 + 显眼「记一笔」
2. **记一笔（表单）** — 全屏或底部大面板，字段少、主按钮大
3. **月统计** — 月份切换 + 三大数字 + 分类占比

底部或顶部导航建议：**今日 | 统计**（记账用 FAB / 主按钮，不必单独 Tab）。

### 4.2 交互原则

- 主路径：打开 → 点记一笔 → 选类型 → 填金额 → 选分类 → 保存 → 回到今日列表
- 金额输入优先数字键盘（移动端 `inputmode="decimal"`）
- 删除需确认；保存成功有轻量反馈（toast / 列表即时出现）
- 空状态要有下一步行动，不要空白页

### 4.3 布局与视觉

- **一套 UI，自适应：** 手机单列；桌面限宽居中（如 max-width ~480–640px 内容柱），避免做成复杂后台仪表盘
- **品牌与氛围：** 简洁清晰即可；避免默认「紫渐变 / 奶油风衬线」等 AI 常见套式；用明确的主色区分收入（如绿）与支出（如红/橙），但不过度装饰
- **首屏职责单一：** 今日页只服务「看今天 + 记一笔」，统计进独立页
- **少卡片：** 列表项用分隔与排版区分即可，非必要不加厚重卡片阴影
- **无障碍：** 按钮可点区域足够大；对比度达标；表单有明确 label

### 4.4 关键文案（示例）

- 记一笔 / 保存 / 删除这条记录？
- 今天还没有记录
- 本月收入 / 本月支出 / 结余

## 5. 技术栈建议

面向学习项目 + 个人使用 + 本地存储，推荐以下方案：

### 5.1 推荐方案（默认）

| 层级 | 技术 | 理由 |
|------|------|------|
| 框架 | **React + Vite + TypeScript** | 生态成熟，适合学习与扩展 |
| 样式 | **Tailwind CSS** | 快速做响应式与统一间距 |
| 路由 | **React Router** | 今日 / 统计 / 编辑页简单路由足够 |
| 状态 | 轻量：`useState` + Context，或 **Zustand** | 数据模型简单，不必上 Redux |
| 持久化 | **localStorage**（或 localforage） | 符合「数据本地存储」；实现成本最低 |
| 图表（可选） | **纯 CSS 进度条** 或 **Recharts** | MVP 可用条形列表代替真图表 |

### 5.2 备选方案

| 方案 | 适用 |
|------|------|
| Next.js | 若后续要加账号/SSR；当前纯本地略重 |
| Vue 3 + Vite | 若更熟 Vue，同等合适 |
| IndexedDB (Dexie) | 记录量很大或要复杂查询时再升级 |

### 5.3 数据层建议

- 单一 `records` 列表 + 预设 `categories` 常量
- 封装 `storage` 模块：`load` / `save` / `export`，UI 不直接操作 `localStorage`
- 金额用「分」整数存储可避免浮点误差（实现时可选；展示仍用元）

### 5.4 工程与质量

- ESLint + Prettier
- 核心逻辑单测：汇总计算、按日/按月过滤、校验
- 部署：任意静态托管（GitHub Pages / Vercel / Cloudflare Pages）

## 6. 非功能性需求

### 6.1 性能

- 首屏可交互目标：常规网络下 < 2s（静态资源 + 本地读数据）
- 单日/单月千级记录内，列表与汇总应无明显卡顿
- 避免每次按键全量重算无关月份；按需按 `date` 过滤

### 6.2 可靠性与数据

- 写入前校验；写入失败时提示用户，不静默丢数据
- `localStorage` 配额不足时给出明确错误与导出建议（v1.2）
- 不承诺跨浏览器/清缓存后仍保留（需在设置或关于页说明）

### 6.3 安全与隐私

- 无账号、无上传：账目数据默认不出本机
- 不引入非必要第三方分析 SDK
- XSS：对备注等文本做安全渲染（React 默认转义即可；避免 `dangerouslySetInnerHTML`）
- 若未来做导出文件，不在文件名中写入敏感路径信息即可

### 6.4 兼容性

- 现代浏览器：Chrome / Safari / Edge / Firefox 近两个大版本
- 移动端优先：iOS Safari、Android Chrome
- 布局：320px～桌面大屏可用；以手机竖屏为设计基准

### 6.5 可用性

- 关键文案简洁；错误可理解（「金额必须大于 0」而非技术堆栈）
- 关键操作可撤销优先用「确认」而非复杂撤销栈（MVP）

### 6.6 可维护性

- 目录按「页面 / 组件 / 数据 / 工具」划分
- 汇总逻辑纯函数，便于测试与复用

## 7. 假设与待确认

写 PRD 时采用了这些默认，若不符可改一版：

1. **分类：** 预设固定分类，MVP 不支持自定义
2. **用户：** 仅个人、单设备本地，无登录
3. **货币：** 仅人民币展示（¥），不做多币种
4. **技术：** React + Vite + TypeScript + Tailwind + localStorage
5. **形态：** 响应式 Web（手机 + 电脑同一套）
